Guide Administrateur — EduLink Sup

Système de gestion pédagogique LMD-CAMES · Version 1.0 · Juin 2026

Ce guide couvre l'ensemble des modules du back-office app.edulink.bj

1. Présentation de EduLink Sup

EduLink Sup est un système de gestion académique conçu spécifiquement pour les universités privées LMD-CAMES du Bénin. Il couvre l'intégralité du cycle académique : de la structuration des programmes à la publication des relevés officiels.

🌐 URLs de production

  • Back-office : app.edulink.bj
  • Portail étudiant : portail.edulink.bj
  • Landing page : edulink.bj

🛠 Stack technique

  • Frontend : React 18 + TypeScript + Vite
  • Backend : Supabase (PostgreSQL + Auth)
  • Déploiement : Vercel
  • Emails : Brevo

2. Accès & connexion

Rendez-vous sur app.edulink.bj et connectez-vous avec votre adresse email et mot de passe.

⚠️ Première connexion : Si c'est votre première connexion, vérifiez votre boîte email pour le lien de confirmation Supabase Auth. Le mot de passe peut être réinitialisé depuis la page de connexion.

Récupération de mot de passe

1
Cliquez sur "Mot de passe oublié" sur la page de connexion
2
Entrez votre adresse email institutionnelle
3
Suivez le lien reçu par email (valable 1 heure)

4. Programmes & Unités d'Enseignement

4.1 Créer un programme LMD

1
Aller dans Programmes & UE → cliquer + Programme
2
Choisir le grade (Licence / Master / Doctorat) — les crédits totaux et la durée se remplissent automatiquement selon CAMES
3
Saisir l'intitulé, la filière, et activer le programme
ℹ️ Standard CAMES : Licence = 180 crédits (3 ans), Master = 120 crédits (2 ans), Doctorat = 180 crédits (3 ans).

4.2 Créer une Unité d'Enseignement (UE)

Depuis l'onglet Unités d'Enseignement, cliquer + UE. Chaque UE requiert :

  • Un code unique (ex : ACG-M1-UE1)
  • Des crédits CECT (en multiples de 3 généralement)
  • La pondération CC / Examen (total doit = 100%)
  • Le semestre d'appartenance

4.3 Ajouter des matières à une UE

Depuis le panneau d'une UE, cliquer + Matière. Chaque matière peut être assignée à un enseignant et avoir un coefficient CM/TD.

5. Semestres & Années académiques

5.1 Créer une année académique

Dans Semestres → onglet Années académiques → + Année. Cocher Année courante pour l'année en cours.

⚠️ Une seule année courante : Marquer une année comme courante désactive automatiquement l'ancienne. Cette valeur est utilisée dans les calculs et l'affichage des résultats.

5.2 Créer un semestre

Cliquer + Semestre et renseigner :

ChampValeur
Libelléex : S1 — Licence GFC 2025-2026
Numéro1 à 8 (LMD = 6 semestres Licence, 4 Master)
Statutplanifié → en cours → clôturé
ProgrammeProgramme LMD associé

5.3 Sessions d'évaluation

Chaque semestre peut avoir deux sessions : normale et rattrapage. Activer le rattrapage depuis le tableau des sessions du semestre.

6. Promotions

Une promotion regroupe les étudiants d'un même niveau et programme pour une année académique donnée.

1
Cliquer + Promotion
2
Sélectionner le programme et le niveau — le nom se génère automatiquement
3
Définir l'effectif maximum et le responsable pédagogique

7. Étudiants

7.1 Ajouter un étudiant

1
Cliquer + Étudiant → remplir le formulaire (Nom, Prénom, Date naissance, Filière, Niveau)
2
Le matricule est généré automatiquement au format hemec-GFC-0031
3
Inscrire l'étudiant à un semestre depuis sa fiche

7.2 Import en masse (CSV/Excel)

Depuis le bouton Importer, téléverser un fichier CSV ou Excel avec les colonnes :

nom, prenom, date_naissance, filiere, niveau, email_auth, telephone
ℹ️ Colonnes obligatoires : nom et prenom. Les autres colonnes sont optionnelles. L'ordre des colonnes n'importe pas — la détection est automatique.

8. Saisie des notes

8.1 Flux de saisie

1
Sélectionner le semestre et la session (normale / rattrapage)
2
Choisir la matière dans la liste — la grille des étudiants s'affiche
3
Saisir les notes CC et Examen — sauvegarde automatique au changement de champ
4
Marquer ABS pour les absents — leur moyenne sera NULL

8.2 Import notes Excel

Cliquer Importer Excel → uploader un fichier avec colonnes matricule, cc, examen. Les notes existantes sont écrasées après confirmation.

8.3 Verrouillage des notes

Après délibération, le responsable pédagogique peut clôturer la session. Les notes ne sont plus modifiables sauf déverrouillage explicite par la direction.

⚠️ Note plancher : Si activé dans Paramètres, une note inférieure au seuil plancher bloque la validation de l'UE même si la moyenne compensatoire est atteinte.

9. Présences

9.1 Créer une séance

Dans Présences → onglet Saisie séance → remplir matière, date, type (CM/TD/TP/Examen) et horaires → cliquer Créer la séance et saisir les présences.

9.2 Saisir les présences

Dans le modal de saisie, cliquer sur le statut de chaque étudiant : ✅ Présent ⏰ Retard ❌ Absent 📄 Justifié. La sauvegarde est immédiate.

9.3 Exclusions pour absences

Le seuil d'exclusion est configurable dans Paramètres → Règles LMD (défaut : 30%). Les étudiants dépassant ce seuil apparaissent dans l'onglet Exclusions. Le bouton Appliquer exclusions auto traite tous les cas en un clic.

🔴 Impact sur les résultats : Un étudiant exclu d'une UE obtient automatiquement une moyenne NULL pour cette UE — il ne peut pas être admis ni compensé sur cette UE.

11. Délibérations

11.1 Processus de délibération

1
Aller dans Délibérations → sélectionner école + semestre
2
Vérifier les décisions calculées automatiquement
3
Modifier manuellement les décisions si le jury décide autrement (override)
4
Cliquer Clôturer le semestre pour verrouiller définitivement
5
Cliquer Publier tous les relevés pour envoyer les relevés aux étudiants
⚠️ Clôture irréversible : Une fois le semestre clôturé, les notes ne peuvent plus être modifiées. Seul le super-admin peut déverrouiller en cas d'erreur.

12. Relevés de notes

12.1 Publication individuelle

Dans Relevés, cliquer Publier en face d'un étudiant. Le relevé est calculé, archivé et envoyé par email si l'option est activée.

12.2 Publication en masse

Cliquer Publier tous pour traiter tous les étudiants sans relevé. Cocher Email à la publication pour envoyer automatiquement.

12.3 Verrouillage

Cliquer 🔓 pour verrouiller un relevé — il devient définitif. Le 🔒 déverrouille. Un relevé verrouillé ne peut pas être republié.

12.4 Blocage pour impayés

Si activé dans Paramètres, un étudiant avec un solde dû supérieur à la tolérance ne peut pas recevoir son relevé. Le solde apparaît en rouge dans la colonne Actions.

13. Enseignants

Gérez le corps enseignant depuis Enseignants. Les grades CAMES disponibles sont : Assistant, Maître-Assistant, Maître de Conférences, Professeur.

Le bouton Matières affiche les matières LMD assignées à chaque enseignant (assignation depuis le module Programmes & UE).

14. Comptabilité

14.1 Créer une facture

Cliquer + Facture → sélectionner l'étudiant, le type de frais, le montant et l'échéance.

14.2 Facturation en masse

Cliquer Facturer une promotion → choisir le semestre et le type de frais → les factures sont créées pour tous les étudiants inscrits (doublons ignorés automatiquement).

14.3 Enregistrer un paiement

Depuis la Fiche d'un étudiant → cliquer + Enregistrer un paiement → saisir le montant et le mode (Espèces / Virement / Mobile Money / Chèque).

14.4 Statuts des factures

15. Paramètres

15.1 Règles LMD CAMES

Depuis Paramètres → ✏️ Modifier dans la section Règles :

  • Seuil validation UE : note minimale pour valider une UE (défaut 10/20)
  • Compensation : active/désactive la compensation inter-UE
  • Note plancher : seuil minimum absolu à l'examen
  • Règle rattrapage : max (meilleure des deux sessions) ou ecrase (le rattrapage remplace)
  • Seuil absences : pourcentage déclenchant une alerte/exclusion (défaut 30%)
  • Blocage relevé : empêche la publication si solde impayé

15.2 Notifications email

Configurer les 3 types de notifications et leur sujet :

  • Publication de relevé : envoyé à chaque publication individuelle
  • Confirmation de paiement : envoyé après enregistrement d'un paiement
  • Alerte absences : envoyé quand le seuil d'absence est dépassé

Variables disponibles dans les sujets : {semestre} {etudiant} {etablissement} {ue}

16. Monitoring (super-admin)

Le dashboard Monitoring est accessible uniquement au super-administrateur réseau. Il affiche :

  • KPIs réseau : écoles actives, étudiants, semestres en cours, relevés publiés
  • Alertes automatiques : impayés critiques (≥ 500 000 FCFA), exclusions en cours (≥ 3)
  • Tableau santé par école avec statut OK / Attention / Critique
  • Liens directs vers Supabase, Vercel et Brevo

L'auto-refresh toutes les 5 minutes peut être activé depuis le bouton en haut de page.

17. Sauvegarde & restauration

17.1 Sauvegarde automatique Supabase

Supabase effectue des sauvegardes automatiques quotidiennes (plan Pro). Pour accéder aux backups :

1
Aller sur supabase.com/dashboard/project/kcfpvnrgutkhakogbjip
2
Section Database → Backups
3
Cliquer Download sur le backup souhaité

17.2 Export manuel

Pour exporter manuellement la base de données :

pg_dump "postgresql://postgres:[PASSWORD]@db.kcfpvnrgutkhakogbjip.supabase.co:5432/postgres" \
  --format=custom \
  --file=edulink_backup_$(date +%Y%m%d).dump
ℹ️ Fréquence recommandée : Export manuel mensuel + backup Supabase automatique quotidien. Conserver les backups manuels dans un stockage externe (Google Drive, OneDrive).

17.3 Restauration

pg_restore --clean --if-exists \
  -d "postgresql://postgres:[PASSWORD]@db.kcfpvnrgutkhakogbjip.supabase.co:5432/postgres" \
  edulink_backup_20260608.dump
🔴 Attention : La restauration écrase toutes les données existantes. Effectuer un export avant toute restauration.

18. FAQ & Dépannage

❓ Le calcul des résultats ne se lance pas

Vérifier que le semestre est en statut en cours et qu'une session normale existe. Aller dans Semestres pour créer la session si nécessaire.

❓ Un relevé ne peut pas être publié

Deux causes possibles :

  • Impayé : le blocage relevé est actif et l'étudiant a un solde dû. Aller dans Comptabilité pour régulariser ou désactiver le blocage dans Paramètres.
  • Session manquante : aucune session normale n'existe pour ce semestre.

❓ La compensation ne s'applique pas

Vérifier dans Paramètres → Règles LMD que Compensation entre UE est activée. Relancer le calcul après activation.

❓ Les emails de relevés ne sont pas reçus

Vérifier :

  • L'adresse email de l'étudiant est renseignée dans sa fiche
  • La notification Publication de relevé est activée dans Paramètres
  • Le tableau de bord Brevo (app.brevo.com) pour voir les logs d'envoi

❓ Comment réinitialiser le mot de passe d'un utilisateur ?

Via le dashboard Supabase → Authentication → Users → sélectionner l'utilisateur → Send password reset. L'utilisateur recevra un email de réinitialisation.

❓ Un étudiant a été exclu par erreur

Aller dans Présences → Exclusions → cliquer Lever en face de l'étudiant concerné. Relancer le calcul des résultats ensuite.
